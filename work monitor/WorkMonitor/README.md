# Work Monitor

**Work Monitor** - небольшая утилита для **macOS 13 и новее**, которая живёт **только в строке меню** (рядом с часами и системными иконками). Отдельного окна в Dock у приложения **нет**: вы открываете компактную панель по клику на иконку и сразу видите картину по машине.

Репозиторий с исходным кодом: [github.com/BrezhnevEugen/WorkMonitor](https://github.com/BrezhnevEugen/WorkMonitor)

## Скачать

Готовый **.dmg** для установки на Mac можно взять в [**Releases**](https://github.com/BrezhnevEugen/WorkMonitor/releases) (удобнее всего [последняя версия](https://github.com/BrezhnevEugen/WorkMonitor/releases/latest)): в блоке **Assets** выберите файл с именем вида `WorkMonitor-2.0.0.dmg` (номер версии может отличаться), откройте образ и перетащите **Work Monitor** в папку «Программы».

При первом запуске macOS может спросить про приложение из интернета - подтвердите в «Системные настройки» - «Конфиденциальность и безопасность», если нужно.

## Зачем он

Часто при разработке и отладке приходится по отдельности смотреть **Docker**, **какие порты слушают процессы**, **сколько съели памяти** и **кто именно жрёт RAM**. Work Monitor **сводит это в одно место** - одна панель из меню, быстрый обзор без переключения между кучей утилит и терминалов.

## Как пользоваться

1. Запустите приложение - в строке меню появится **монохромная иконка** (кружок рядом с числом открытых портов).
2. **Клик** по иконке открывает **всплывающую панель** с обзором.
3. Панель **сама обновляет данные каждые 2 секунды**, пока открыта; в фоне данные обновляются реже (чтобы счётчик в меню оставался актуальным, но система не грузилась).
4. Сверху панели - **шестерёнка** для настроек, внизу - **About** (о программе) и выход (**Quit**).

Никакой сложной настройки: установили или запустили из `.app` - иконка уже в меню.

## Что внутри панели

### Сводка (CPU / RAM / Disk / Net)

Сверху панели - **полоса метрик**: загрузка процессора, занятая оперативная память, свободное место на диске и текущая скорость сети (скачивание). У каждого показателя мини-индикатор: зелёный - норма, жёлтый - высокий расход, красный - критично.

### Иконка в меню-баре

Значок **монохромный** и автоматически меняет форму по состоянию системы: заполненный кружок - всё в норме, пустой кружок - ничего не слушается, треугольник - предупреждение (память выше 75% или есть unhealthy Docker-контейнер), восклицательный знак в круге - критично (память выше 90% или Docker пропал). Рядом с иконкой - **счётчик открытых портов**. macOS сам подкрашивает значок под текущий фон строки меню (белый на тёмной, чёрный на светлой).

### Порты

Какие **TCP-порты** сейчас в состоянии **прослушивания**, к какому **процессу** они относятся, на каком **адресе** слушают. По умолчанию список **сгруппирован по приложениям** (одно приложение может слушать несколько портов - например, node-воркер + дебаггер), группы сворачиваются. Если на порту отвечает похожий на веб сервис, утилита может **подсказать** и дать открыть страницу в браузере (проверка идёт на `localhost`). При наведении на строку появляются кнопки **open** (открыть в браузере) и **kill** (мягко завершить процесс).

### Docker

Если **Docker** у вас установлен и демон запущен, показывается список **контейнеров** (статус, образ, проброс портов, время работы). Статус отмечен цветной точкой: зелёная - running, жёлтая - unhealthy или restarting, серая - остановлен. Если Docker недоступен, выводится понятное сообщение, а не «молчаливый» пустой блок.

### Процессы

**Топ-6** процессов по потреблению памяти - видно, кто съел RAM. Горячие процессы (>10% памяти) подсвечиваются.

## Настройки

Открываются шестерёнкой в верхнем правом углу панели.

**Тема.** Четыре варианта: `Авто (по системе)` - следит за macOS и переключается между тёмной и светлой автоматически, когда вы меняете режим системы; `Тёмная терминальная` - тёмная с зелёным акцентом (по умолчанию при первом запуске); `Светлая минималистичная` - светлая; `Полночь` - синяя тёмная. Рядом с каждым вариантом показаны три цветных свотча - видно палитру до переключения.

**Отображать разделы.** Чекбоксы **Порты** / **Docker** / **Процессы** - любой раздел можно скрыть, если он вам сейчас не нужен.

**Порты → Группировать по приложению.** Когда включено, порты одного процесса собраны в сворачиваемую подгруппу; когда выключено - плоский список.

**Язык.** Системный / English / Русский. Меняется сразу, без перезапуска.

**Система.** Кнопка, которая открывает системный экран Login Items, чтобы настроить автозапуск при входе в систему.

Все настройки сохраняются между запусками.

## Как это устроено «под капотом» (коротко, по смыслу)

Приложение **не заменяет** системные мониторы целиком - оно **считывает** уже известные macOS и окружению данные (отчёты о памяти через `vm_stat`, список слушающих сокетов через `lsof`, процессор через `top`, диск через `df`, сеть через `netstat`, при наличии - вывод `docker ps`) и **показывает их в одном интерфейсе**. Так вы быстрее понимаете, **что сейчас происходит** на машине, а не копаетесь по отдельности.

## Требования

- **macOS 13** (Ventura) или новее.
- (опционально) Установленный и запущенный **Docker Desktop** - если хотите видеть контейнеры.
